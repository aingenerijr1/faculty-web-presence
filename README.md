# Faculty Course Support Hub

## Project Purpose

The Faculty Course Support Hub is a simple two-page faculty web presence designed for Professor Dawn Ellis. The site helps students quickly find instructor information, office hours, course resources, and a way to request support.

The problem this project addresses is that students often need one clear place to locate important faculty information and communicate questions. This website provides a basic structure for organizing that information in a simple and accessible way.

## Repository Contents

This repository includes the following files:

* `index.html` - The front-facing faculty homepage with instructor information, office hours, courses taught, and student resources.
* `contact.html` - A student support form page where students can enter their name, email, course, reason for contact, and message.
* `README.md` - This file, which explains the purpose of the project and how to view the HTML pages.
* `assets/` - A folder reserved for supporting files such as images.

## How to View the Pages Locally

1. Download or clone this repository to your computer.
2. Open the project folder.
3. Double-click `index.html` to open the homepage in a web browser.
4. Use the navigation links to move between the homepage and the contact page.

## Design Notes

The visual design document was used as a blueprint for the HTML prototype. The homepage wireframe included a header with Dawn Ellis's name, navigation, hero section, office hours, courses taught, student resources, and footer. The contact page wireframe included a header, navigation, form introduction, input fields, submit button, and footer.

The HTML structure follows the same layout as the wireframe so that the prototype matches the original design plan.

## Reflection

This project focuses on design thinking, structure, and basic web development workflow. The main goal was not to create a fully styled or functional application, but to demonstrate how a simple user interface can be planned, prototyped in HTML, and managed through GitHub.
